### 3.3 Execution & Slippage Model

| Component | Description |
|-----------|-------------|
| Slippage estimation | Order book depth walk — `estimate_slippage_bps(symbol, qty, side)` |
| Execution | Matched against limit order book (`OrderBookManager`) |
| Market orders | Immediate execution against best available limit prices |
| Limit orders | Price-time priority queue; filled, partially filled, or open |
| IOC orders | Immediate fill of available liquidity; remainder cancelled |
| FOK orders | Full fill or full cancel |
| Partial fills | Supported for IOC and market orders |
| Order cancellation | `POST /v1/cancel` — open limit orders only |

**Slippage estimation from order book depth:**

```python
def estimate_slippage_bps(self, symbol, quantity, side):
    levels = ob["asks"] if side == "buy" else ob["bids"]
    remaining = abs(quantity)
    total_cost = 0.0
    for level in levels:
        if remaining <= 0: break
        fill = min(remaining, level["quantity"])
        total_cost += fill * level["price"]
        remaining -= fill
    if remaining > 0:
        total_cost += remaining * mid * 1.05  # 5% penalty beyond book
    avg_price = total_cost / abs(quantity)
    slippage_bps = (avg_price - mid) / mid * 10000
    return abs(round(slippage_bps, 2))
```

## 6. Known Limitations

1. **Synthetic data only** — Market dynamics follow a Gaussian random walk; no regime shifts, jumps, or fat tails characteristic of real markets.
2. **No partial fills** — All orders execute at full quantity or are rejected.
3. **Limited symbol set** — 5 correlated synthetic assets; no portfolio diversification effects beyond this set.
4. **No latency model (Phase 1–2)** — Execution is effectively instantaneous until Phase 3 introduces delay simulation.
5. **No market impact** — Trade size does not affect price beyond available order book depth (15 levels).
