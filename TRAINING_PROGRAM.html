    ⚙ AST Auditor

    What Gets Checked

    The AST auditor parses your Python code into an abstract syntax tree and runs 9 pattern-matching checks. Every EOD evaluation runs through this gauntlet.

    Critical −40
    NO_NETWORK_EXCEPT
    urllib.request.urlopen() or Request() without try. Exchange goes down = your pipeline crashes.
    Critical −40
    EVAL_EXEC
    eval(...) or exec(...) detected. Arbitrary code execution is a security risk and SOCPA violation.
    Critical −40
    PROHIBITED_IMPORT
    pickle, shelve, marshal — deserialization vulnerabilities that compromise audit integrity.
    Warning −10
    NESTED_LOOP
    for i in range(n): for j in range(m): — O(n²) complexity that doesn't scale to production datasets.
    Warning −10
    DIVISION_BY_ZERO
    a / b without a b != 0 guard. Runtime crash waiting to happen.
    Warning −10
    INFINITE_WHILE
    while True: without a break statement. Never-terminating loop that hangs the pipeline.
    Warning −10
    GLOBAL_STATE
    global x at module level. Side effects make code untestable and unpredictable.
    Info −0
    MISSING_ANNOTATION
    Function parameter or return value without a type hint. PEP 484 compliance is expected.
